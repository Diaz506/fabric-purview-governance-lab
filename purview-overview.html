02Key concepts & terminology

Purview vocabulary can be confusing because the product is evolving from the classic Data Map experience to the new Unified Catalog. These are the terms you'll hear.

Governance domain — In the new Microsoft Purview Unified Catalog, a governance domain is the top-level organizing boundary for governance: it groups business glossary terms, data products, owners/stewards, and policies for a business area (e.g. “Lumina Retail” or “Finance”). Think of it as “who owns this slice of governance and what vocabulary applies here.” It is a business/ownership boundary — distinct from a collection, which is the technical boundary used by the Data Map.
🗺️

Data Map

The technical foundation: a live map of scanned assets, their schema, and metadata across sources.

🗂️

Collection

A hierarchical container in the Data Map used to organize sources and apply technical RBAC permissions.

🔌

Data source

A registered system Purview connects to (Fabric, Azure SQL, ADLS, Snowflake…). Fabric is registered once per tenant.

🔍

Scan & scan rule set

A scan harvests metadata and runs classifications. A scan rule set bundles which classifications (built-in + custom) to apply.

🏷️

Classification

A tag applied to a column/asset indicating what the data is (Email, Credit Card, custom Loyalty Number). Auto-applied during scan.

🔒

Sensitivity label

A protection label (Confidential, Highly Confidential) from Purview Information Protection. Different from a classification — labels enforce protection, classifications describe content.

📖

Glossary term

A governed business definition (e.g. Loyalty Number) with steward, status, hierarchy, and synonyms — linkable to technical columns.

📦

Data product

In Unified Catalog, a curated, published bundle of data assets governed within a domain for consumers to discover and request access.

🧩

Asset

Any discovered object: a table, file, column, report, or dataset. Assets carry schema, classifications, lineage, and term links.

03What Purview covers

Capabilities the platform provides out of the box (the ones this lab exercises are marked).

🗺️

Automated scanning of Fabric, Azure, multicloud, and SaaS sources; harvests schema and technical metadata into a searchable map.

📖

Business glossary in lab

Hierarchical terms with synonyms, acronyms, stewards, approval status, and bulk CSV import. Link terms to physical columns.

🏷️

Classifications in lab

200+ built-in classifiers (PII, financial, health) plus custom regex rules and thresholds for org-specific identifiers.

🔗

End-to-end visualization of how data moves and transforms across pipelines, dataflows, and reports; assess change impact.

✅

Data quality

Define rules (completeness, uniqueness, validity), profile data, and track quality scorecards over time within a domain.

🔎

Unified Catalog & search

A single search surface across assets, glossary, lineage, and quality — with governance domains and data products.

🔒

Information protection

Sensitivity labels and label-based protection that travel with the data, integrated with Microsoft 365.

🛡️

Access & DLP policies

Policy-based access provisioning and data loss prevention scoped by classification and source.

🤖

AI-assisted governance

Suggestions for classification, glossary term linking, and descriptions to speed up curation at scale.

04Beyond Purview: the broader data dictionary

Purview is a governance and catalog platform, not a full data dictionary / data modeling tool. A complete enterprise data dictionary describes the precise semantics and rules of every data element — much of which Purview only partially covers or leaves to other tools.

✔ What a catalog like Purview does well

  • Discover & inventory physical assets automatically
  • Tag sensitive data and enforce governance policy
  • Provide business definitions and ownership
  • Show lineage and search across the estate

✘ What a data dictionary additionally needs

  • Precise field-level semantics: data type, format, units, precision
  • Allowable values / code sets / reference data with meanings
  • Validation, derivation, and calculation rules
  • Conceptual & logical models (entities, keys, cardinality)
  • Versioned definition history with approval workflow
  • Canonical / source-of-truth mapping across systems

05Putting it together

Use Purview as the governance backbone, and complement it where a full data dictionary is required.

🏛️

Govern with Purview

Catalog, classify, link glossary terms, and enforce policy across Fabric and the wider estate.

📐

Model elsewhere

Capture entities, keys, and logical models in a modeling tool (erwin, ER/Studio, dbt, SqlDBM) and reference them from the glossary.

📋

Manage reference data

Keep code sets and allowable values in an MDM/reference-data service; surface the meaning via glossary terms.

⚙️

Enforce rules in pipelines

Implement validation and derivation logic in Fabric pipelines/notebooks; document the intent in the dictionary.

Rule of thumb: Purview answers “what data do we have, what is it, who owns it, and is it sensitive?” A data dictionary additionally answers “exactly how is each field defined, what values are valid, and how is it derived?” You typically need both.
